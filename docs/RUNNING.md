# Как запустить проект

Четыре способа. Выбирайте по задаче:

| Задача | Способ | Раздел |
|---|---|---|
| Показать проект, поработать вживую | **На своей машине** | 1 |
| Развернуть на сервере | **Docker** | 2 |
| Построить индекс, посчитать метрики | **CLI** | 3 |
| Проверить, что ничего не сломано | **Тесты** | 4 |

Главное правило: **одновременно на одной машине поднимайте что-то одно.** Папка
`data/` общая, и два процесса писали бы в одни файлы индекса. Плюс Telegram отдаёт
обновления только одному потребителю на токен — два бота будут отбирать сообщения
друг у друга.

## 0. Что нужно один раз

```powershell
cd C:\Users\psavd\Downloads\projTaym\Taymazi-Yakutyanochki
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install torch -r requirements-web.txt
.\.venv\Scripts\python.exe -m pip install -r requirements-dev.txt   # если нужны тесты
```

Затем скопируйте `.env.example` в `.env` и заполните. Минимум для веба — ничего
(всё работает на умолчаниях). Для бота нужны две строки:

```
TELEGRAM_BOT_TOKEN=<токен от @BotFather>
SERVICE_TOKEN=<любая длинная случайная строка>
```

`SERVICE_TOKEN` — это внутренний пароль бота к API. Пока он не задан, ручек
`/api/bot` не существует вовсе, и бот работать не будет.

**Для групповых чатов обязательно** выполните у [@BotFather](https://t.me/BotFather)
`/setprivacy` → **Disable**. По умолчанию Telegram показывает боту только команды,
и присланных в группу фотографий он просто не увидит.

## 1. На своей машине — для демонстрации

Основной способ для показа: **добавление фотографий здесь примерно в 11 раз
быстрее**, чем в контейнере (0,6 с против 6,8 с на снимок). Причина не в модели,
а в записи файлов: в Docker папка `data/` подключена с Windows через виртуальную
машину, и запись мелких файлов там дорога. Поиск при этом почти не отличается —
82 мс против 119 мс.

```powershell
docker compose down          # если контейнеры были подняты
.\scripts\run_local.ps1
```

На Linux и macOS — `./scripts/run_local.sh`.

Откроются три окна: API, фронтенд, бот. Интерфейс — **http://localhost:5173**.

| Ключ | Что делает |
|---|---|
| `-NoBot` | без Telegram-бота |
| `-ApiOnly` | только API, без фронтенда |
| `-ApiPort 8001` | другой порт, если 8000 занят |

Первые 15–30 секунд API загружает модель. Готовность видна по строке
`Модель готова: dim=512` в его окне; до этого страница ответит ошибкой.

Останавливать — Ctrl+C в каждом окне.

### Если нужен контроль, те же три команды вручную

```powershell
.\.venv\Scripts\python.exe -m uvicorn web.app:app --port 8000     # API
npm --prefix web-ui run dev                                        # интерфейс :5173
.\.venv\Scripts\python.exe -m bot.bot                              # бот
```

При работе над кодом запускайте API с `--reload`: без него uvicorn не перечитывает
изменённые файлы, и вы будете отлаживать старую версию, не понимая, почему правки
не действуют.

## 2. Docker — для сервера

```powershell
docker compose up -d --build
```

Три сервиса: `web` (API и CLIP), `ui` (nginx со статикой), `bot`. Интерфейс —
**http://localhost:8080**, порт задаётся `WEB_PORT`.

```powershell
docker compose ps                 # состояние и healthcheck
docker compose logs -f web        # лог API
docker compose logs -f bot        # лог бота
docker compose down               # остановить всё
```

Первая сборка занимает 10–20 минут: качается torch (~200 МБ) и веса CLIP (~600 МБ).
Последующие сборки при изменении кода — секунды, потому что тяжёлые слои кэшируются.

Что должно быть в логе `web` при успешном старте:

```
Модель готова: dim=512, device=cpu
Демо-база подключена: /app/index
Данные: /app/data | регистрация: открыта | ... | ручки бота: вкл
```

Строка `ручки бота: выкл (нет SERVICE_TOKEN)` означает, что токен не долетел
из `.env` — бот работать не будет.

### Только веб, без бота

```powershell
docker compose up -d web ui
```

### Развёртывание на настоящем сервере

- **TLS** здесь не терминируется: поставьте перед nginx certbot или traefik либо
  пробросьте 443 с хоста. Как только `PUBLIC_URL` начинается с `https://`,
  приложение само включает флаг `Secure` у cookie сессии.
- **`TRUST_PROXY=1`** уже задан в `docker-compose.yml` и нужен обязательно.
  Ограничение частоты входа считает попытки по IP, а за nginx все запросы приходят
  с одного адреса: без этого флага пятая неудачная попытка любого пользователя
  закрыла бы вход всем остальным.
- **Порт API наружу не публикуется** — снаружи доступен только nginx, а ручки
  `/api/bot/*` он отдаёт как 404: бот ходит к API по внутренней сети.

## 3. CLI — индекс и метрики

Веб и бот не строят индекс COCO — это делает CLI. Подробности о датасете
в README, раздел 5.

```powershell
# построить индекс (один раз)
.\.venv\Scripts\python.exe src\clip_zero_shot_search.py build `
    --images_dir .\data\images --captions_csv .\data\captions.csv --index_dir .\index

# поиск по тексту
.\.venv\Scripts\python.exe src\clip_zero_shot_search.py text `
    --index_dir .\index --query "собака играет в снегу" --translate --top_k 5

# метрики Recall@1/5/10 и время ответа
.\.venv\Scripts\python.exe scripts\eval_recall.py --index_dir .\index --latency
```

Построенный `index/` подключается к вебу и боту как **демо-база только для чтения**:
это единственное место, где работает поиск «фото → подпись», потому что у
пользовательских баз подписей нет.

## 4. Тесты

```powershell
.\.venv\Scripts\python.exe -m pytest tests -q
```

188 тестов, около полутора минут. Веса CLIP при этом не загружаются: модель
подменяется детерминированной заглушкой, поэтому тесты быстрые и не зависят от сети.

## 5. Что делать, если не работает

| Симптом | Причина и что делать |
|---|---|
| На `localhost:5173` пусто | Фронтенд не поднялся. Посмотрите его окно; проверьте, что порт слушается: `Get-NetTCPConnection -LocalPort 5173` |
| На `localhost:8080` пусто | Контейнеры не запущены: `docker compose ps` |
| Страница отвечает ошибкой первые полминуты | API ещё грузит модель — дождитесь `Модель готова` |
| Правки в коде не действуют | uvicorn запущен без `--reload`, либо в Docker нужен `docker compose build` |
| Вход не проходит, хотя пароль верный | Пять неудачных попыток подряд блокируют вход на 15 минут |
| Пароль забыт | `.\.venv\Scripts\python.exe -m web.admin reset-password --login <логин> --password <новый>` |
| Бот молчит в группе | Не выполнен `/setprivacy` → Disable у @BotFather |
| Бот не отвечает нигде | Проверьте `SERVICE_TOKEN` в `.env` и лог: `docker compose logs bot` |
| Бот отвечает через раз | Запущено несколько экземпляров: в логе `TelegramConflictError`. Надёжнее всего снять все и поднять заново: `Get-CimInstance Win32_Process -Filter "Name='python.exe'" \| Where-Object { $_.CommandLine -like "*bot.bot*" } \| ForEach-Object { Stop-Process -Id $_.ProcessId -Force }`, затем `.\scripts\run_local.ps1`. Считать экземпляры по числу процессов нельзя: один живой бот — это два `python.exe` |
| Бот отвечает «Не знаю такой команды» | Так и задумано: неизвестные команды не уходят в поиск |
| Скрипт `.ps1` не запускается | Файл скачан из интернета: `Unblock-File .\scripts\run_local.ps1` |
| Ошибки про два процесса или битый индекс | Подняты и Docker, и локальный запуск одновременно. Оставьте что-то одно |

## 6. Что где лежит

| Что | Где |
|---|---|
| Базы пользователей и чатов | `data/users/<user_id>/databases/<db_id>/` |
| Учётные записи, сессии, задачи | `data/app.db` (SQLite) |
| Индекс COCO для демо-базы | `index/` |
| Настройки | `.env`, образец — `.env.example` |

Данные общие для всех способов запуска: база, набранная через Docker, видна и при
локальном запуске, и в боте, и на сайте. Для демонстрации это удобно — можно
набрать снимки ботом и показать их в браузере.
